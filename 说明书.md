# 3D 打印实验室网站 Demo — 说明书

## 一、我做了什么

### 1. 技术选型

- **前端框架**：React 18 + TypeScript  
- **构建工具**：Vite 5  
- **路由**：React Router 6  
- **样式**：CSS Modules（无 UI 库，便于后续自定义和部署）  
- **字体**：Noto Sans SC（Google Fonts），兼顾中文与科技感  

整体是**纯前端静态站点**：开发时在本地运行，构建后得到静态文件（`dist/`），可部署到任意云服务器（Nginx、OSS、Vercel 等），无需后端即可先跑通整站。

### 3. 功能与页面说明

| 页面 | 路径 | 说明 |
|------|------|------|
| 主页 | `/` | 参考 Apple 官网风格：大标题、副标题、居中 3D 打印机 CSS 动画，下方五个入口卡片跳转子页 |

- **风格**：浅色背景、简洁排版、圆角卡片、蓝色强调色，兼顾清新与科技感，便于后续接云服务器后继续扩展。

## 二、如何在本地打开并运行

### 环境要求

- 已安装 **Node.js**（建议 18 或 20，可从 [nodejs.org](https://nodejs.org) 下载）

### 步骤

1. **进入项目目录**  
   在终端中执行：
   ```bash
   cd d:\Pycharm_Data\3d_print_web\3d_print_web
   ```

2. **安装依赖**  
   ```bash
   npm install
   ```

3. **启动开发服务器**  
   ```bash
   npm run dev
   ```
   终端会输出本地访问地址，一般为：  
   **http://localhost:5173**  
   用浏览器打开该地址即可查看网站。

4. **（可选）构建用于部署**  
   若之后要部署到云服务器，可先本地构建：
   ```bash
   npm run build
   ```
   构建产物在 `dist/` 目录，将其中全部文件上传到服务器静态目录（如 Nginx 的 `root` 或 OSS 桶）即可。  
   若部署在子路径（如 `https://你的域名/lab/`），需在 `vite.config.ts` 中把 `base` 改为 `'/lab/'` 后重新执行 `npm run build`。

## 三、后续可扩展方向

- 在对应子页面中替换占位内容，加入真实教程、文件列表、论坛接口等。  
- 接入云服务器时：用 Nginx/Apache 托管 `dist/`，或使用对象存储 + CDN；若需登录、提交打印等，再增加后端 API 与前端请求即可。  
- 若需 SEO，可考虑后续迁移到 Next.js 等支持服务端渲染的框架，当前 Demo 已满足「本地可运行、结构清晰、便于接云」的目标。

如有具体子页面内容或接口设计需求，可以继续说明，我再按你的指示补充或修改。
